ルート地図（OpenLayers版）

表示例

設置方法

このページで説明するOpenLayers版ルート地図のソースコードを利用する際には、npm（パッケージ管理システム）が必要です。また、本サイトで説明している各種ルート地図やGPX2GeoJSONのソースコードはGitHubで管理しているので、ソースコードの入手にはgit（バージョン管理システム）があると便利です。Windows 10でnpmやgitを使うには、Windows Subsystem for Linux（WSL）のUbuntuを利用するのが最も手軽です。UbuntuはMicrosoftストアから無料でインストールできます。Ubuntuを起動してBashシェルの窓を開いたら、次のコマンドを実行します。

コマンド実行：Ubuntuアップデート
sudo apt update
sudo apt upgrade
コマンド実行：npmインストール
sudo apt install npm

以降のコマンドもBashシェルの中で実行します。

ページ内埋め込み

表示例のようなOpenLayers版ルート地図を表示するには、対象サイトに下記のディレクトリ構成図のようにファイルを設置し、対象ページに表示のためのコードを挿入します。ここで、ファイル名中の'*'は、npmによってランダムに割り振られた英数字列です。また、ファイルomap_ex2.〜は、npmを実行して得られる出力ファイルです。

ディレクトリ構成図
.
├── your_page.html（対象ページ）
├── omap_ex2.html
├── omap_ex2.*.js
├── omap_ex2.*.js.map
├── omap_ex2.*.css
├── omap_ex2.*.css.map
└── example
    └── routemap.geojson
対象ページに挿入するコード
<iframe width="640" height="480" style="border:none;"
 src="omap_ex2.html?lat=35.913163&lon=138.721400&zoom=15&url=example/routemap.geojson">
</iframe>

その他の表示方式（別窓ポップアップ）については、設置方法（共通）を参照して下さい。

ソースコード

以下のgitコマンドでルート地図のソースコード一式を入手すると、ディレクトリ構成図のファイルが得られます。このうち、ディレクトリnode_modulesは'npm update'を実行すると作成され、その中に必要なモジュールがインストールされます。

コマンド実行：ソースコードの入手
git clone https://github.com/anineco/gps-track-maps.git
コマンド実行：モジュールのインストール
cd gps-track-maps
npm update
ディレクトリ構成図（OpenLayers関係のみ抜粋）
gps-track-maps
├── docs/
│   └── example/
│       └── routemap.geojson （GeoJSONファイル）
├── node_modules/
├── package-lock.json
├── package.json
└── src/
    ├── omap_ex2.html
    ├── omap_ex2.js
    ├── Popup.*
    └── Toolbar.*

変更後、'npm start'を実行し、表示されるURL（'localhost:1234'）をブラウザで閲覧して、ルート地図を確認します（注：'omap_ex2.html'を閲覧する場合は、'localhost:1234/omap_ex2.html'を開く）。




コマンド実行：テスト


npm start




表示を確認してOKならば、'npm run build'を実行して、ウェブ公開用のファイルを生成します。生成ファイルをウェブサイトに設置し、閲覧してOKならば作業完了です。




コマンド実行：ビルド


npm run clean
npm run build
